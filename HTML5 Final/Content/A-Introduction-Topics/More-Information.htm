Accessibility in web design

Digital material on the web can also be designed to be accessible for everyone. It can be used on any device even if it is different from most devices. According to HTML & CSS: Design and Build Websites by Jon Duckett, web material is now being made more accessible to people who use a variety of digital devices “including desktop computers, laptops, tablets and mobile phones." One may choose to use a certain device as they are most comfortable using that particular device rather than something else.

Kalbag directly applies universal design to web design in Accessibility For Everyone. She compares accessible web design to having an option on a website to increase the font size, but it is simply “another element crowding a page layout, and people have to go out of their way to find it,” which might actually make it harder for some people to read as they have to locate the option. Conversely, universal web design would entail giving the whole webpage a readable font size, meaning that anybody can read the webpage with ease and without having to locate options to make it accessible.

There are several key practices to making text universally accessible.

Alternative text is useful for more than one situation. For example, if you are blind, it can give you a description of what the image is. However, that is not a blanket case.

Semantic heading levels

Semantic heading levels allow for hierarchy in text, drawing people’s attention to what a topic is about and what subtopics, if any, there are. The highest heading rank is usually <h1>, which is the main topic’s heading, followed by <h2> for a subtopic, <h3> for a subtopic of that subtopic, and so on until <h6>. The World Wide Web Consortium describes headings as being necessary to organize where certain information located in a text belongs.

It is not good practice to skip from <h1> to <h3> and then to <h2>, as that makes it appear that the information directly under <h2> is a subtopic of the information directly under <h3>. An exception is when closing an <h3> subtopic and starting a new <h2> subtopic.

An effective use of semantic heading levels is:

<h1>Accessibility in web design</h1>

<h2>Accessibility: accessible design versus universal design</h2>

<h3>Accessible design</h3>

<h3>Universal design</h3>

<h2>Accessibility in web design</h2>

Visual hierarchy and contrast

If a web page has a good visual hierarchy in how it presents its text, and contrasts different modes of material well against each other, it is accessible. If the most important material is made to stand out effectively - so it can be accessed easily - it can be useful, but it should be used with care. As per Duckett, "...if too many aspects are competing for your attention, it can be hard to find the key messages."
